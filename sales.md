## FRAMEWORK 1 — SPIN Selling (상담 대화 설계)

Neil Rackham의 SPIN은 컨설팅 영업의 교과서. 고압적 클로징 대신 **고객 스스로 필요성을 깨닫게** 하는 구조.

### 4단계 질문 구조

```
S — Situation (현황 파악)
"지금 연금은 어떻게 준비하고 계세요?"
"직장 복지로 퇴직연금이 있으신가요?"

P — Problem (문제 인식)
"나중에 받으실 때 세금이 어떻게 되는지 아세요?"
"지금 이 금액으로 60세 이후 생활비가 충분할 것 같으세요?"

I — Implication (문제의 파급 인식)
"그게 해결 안 되면 은퇴 후 매달 얼마나 부족할 것 같으세요?"
"지금 이 구조 그대로 10년 지나면 어떻게 될까요?"

N — Need-payoff (해결책 가치 확인)
"만약 세금 없이, 복리로 자동 적립되는 구조가 있다면 어떨 것 같으세요?"
"이걸 지금 설계해두면 앞으로 신경 안 써도 된다면 좋으시겠죠?"
```

### JW Financial 적용 원칙
- S → P → I → N 순서 절대 바꾸지 않는다
- 각 단계에서 **고객이 말하게** 만든다 (진우님은 듣는다)
- I 단계가 클로징 성패를 가른다 — 충분히 시간 투자

## FRAMEWORK 2 — $100M Offers (제안 설계)

Alex Hormozi: "가격 저항은 가치가 안 보여서다. 가격을 내리지 말고 가치를 올려라."

### 가치 방정식
```
가치 = (원하는 결과 × 달성 가능성) / (시간 × 노력)

↑ 올려야 할 것: 결과의 크기, 달성 가능성
↓ 줄여야 할 것: 걸리는 시간, 고객의 수고
```

### JW Financial 제안 설계 적용

| 요소 | 일반 설계사 | JW Financial |
|------|------------|--------------|
| 원하는 결과 | "노후 대비" (모호) | "63세부터 월 300만원 수령" (구체적) |
| 달성 가능성 | 상품 설명 | 시뮬레이션 수치 + 사례 |
| 시간 단축 | 고객이 알아서 | 진우님이 세팅·관리 대행 |
| 노력 절감 | 고객이 공부해야 | 카드뉴스·월별 리포트 제공 |

### 제안서 필수 구성 요소
1. **꿈의 결과** — 숫자로 표현 (월 XX만원, XX세 은퇴)
2. **현재와의 갭** — 지금 안 하면 얼마나 손해인가
3. **왜 지금인가** — 복리, 세제혜택 기한, 나이 요인
4. **왜 JW인가** — 1:1 관리, 모니터링, 지속 커뮤니케이션
5. **리스크 제거** — "언제든 조정 가능", "중도해지 시 어떻게 되는지"

## FRAMEWORK 3 — Psychology of Selling (심리 기반 클로징)

Brian Tracy: "구매 결정은 감정으로 하고, 논리로 정당화한다."

### 주요 심리 패턴 & 대응

**"생각해볼게요"**
```
→ 실제 의미: 아직 가치를 못 느꼈거나, 숨겨진 반대 의견이 있다
→ 대응: "어떤 부분이 가장 걸리세요?" (직접 물어보기)
→ 절대 금지: "그럼 연락 주세요" (= 계약 포기)
```

**"남편/아내랑 상의해봐야 해요"**
```
→ 대응: "같이 오시면 제가 설명드릴 수 있어요. 언제가 편하세요?"
→ 또는: "배우자분이 가장 궁금해하실 부분이 뭘까요? 미리 정리해드릴게요"
```

**"지금 여유가 없어요"**
```
→ 대응: "지금 부담 없는 최소 금액으로 시작하고, 여유 생기면 올리는 구조도 있어요"
→ 핵심: 금액 논쟁 말고 '시작 자체'의 가치로 전환
```

**"다른 데랑 비교해보려고요"**
```
→ 대응: "비교해보시는 거 당연하죠. 어떤 기준으로 보실 건지 알면 제가 미리 정리해드릴 수 있어요"
→ 목적: 비교 기준을 JW Financial에 유리하게 설정
```

## JW Financial 영업 단계별 SOP

### 1차 미팅 — 신뢰 구축
- 목표: 계약 아님, **다음 미팅 약속**
- SPIN의 S-P 단계만 진행
- 끝에 반드시: "다음에 시뮬레이션 보여드릴게요, 언제 시간 되세요?"

### 2차 미팅 — 가치 설계
- 시뮬레이션 지참 (숫자로 보여주기)
- SPIN의 I-N 단계
- Hormozi 가치 방정식 적용한 제안서
- 끝에: "이 방향으로 설계해드릴까요?" (Yes 유도)

### 3차 미팅 — 클로징
- 2차에서 Yes 받은 내용 실행
- Tracy의 심리 대응 준비
- 계약 완료 후 즉시 온보딩 시퀀스 시작

### 이탈 방지 (2차→3차 사이 — 가장 위험 구간)
- D+1: 보험 원리 교육 카카오 메시지 (판매 언어 금지)
- D+2: 비식별 성공 사례 공유
- D+3: "3차 미팅 이렇게 하면 됩니다" 안내

## 작업 요청 유형별 출력 형식

| 요청 | 출력 형식 |
|------|-----------|
| `1차 미팅 준비해줘` | SPIN S-P 질문 리스트 + 미팅 흐름 |
| `2차완료 [고객명]` | 이탈방지 D+1~D+3 카카오 메시지 3종 |
| `계약완료 [고객명]` | 온보딩 D+0/D+7/D+30/D+90 메시지 세트 |
| `[고객명] 거절 처리` | Tracy 심리 대응 스크립트 |
| `제안서 초안` | Hormozi 5요소 구조 제안서 |
| `소개 요청 메시지` | 성장스토리 + 타겟 명확화 + 방법 구체화 |
