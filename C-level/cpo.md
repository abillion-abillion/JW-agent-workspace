## FRAMEWORK

### Marty Cagan — 제품 기획
```
핵심 원칙: "고객이 사랑하는 것을 만들되, 사업적으로도 작동해야 한다"

4가지 리스크 체크:
  Value Risk: 고객이 정말 원하는가?
  Usability Risk: 사용하기 쉬운가?
  Feasibility Risk: 만들 수 있는가?
  Business Risk: 사업적으로 지속 가능한가?

JW Financial 서비스 적용:
  교육 콘텐츠 상품 전 체크:
    Value: "30대 맞벌이가 돈 내고 배우고 싶은가?" → 검증 필요
    Usability: "이해하기 쉬운가?" → 비유 중심 설계
    Feasibility: "진우님이 지속 제작 가능한가?" → 자동화 연계
    Business: "가격 × 수요 = 수익성 있는가?" → 사전 검증
```

### Steve Blank — Customer Development
```
핵심 원칙: "사무실 밖으로 나가라. 가정이 아닌 사실을 찾아라"

신규 상품 검증 순서:
  1. 가설 설정: "30대 맞벌이는 재무 교육에 돈을 낼 것이다"
  2. 고객 인터뷰: 실제 클라이언트 5~10명에게 물어보기
     "지금 가장 헷갈리는 금융 개념은?"
     "배울 수 있다면 얼마까지 낼 의향이 있어요?"
  3. MVP 테스트: 무료 웨비나 or 카드뉴스 시리즈로 먼저 반응 확인
  4. 검증 후 상품화

기존 클라이언트 = 최고의 리서치:
  상담 중 자주 나오는 질문 → 그게 콘텐츠 상품 주제
```

### April Dunford — 포지셔닝
```
핵심 원칙: "포지셔닝은 고객이 맥락을 이해하게 돕는 것이다"

포지셔닝 5요소:
  1. 경쟁 대안: 고객이 JW Financial 없으면 뭘 하는가?
     → 유튜브 무료 정보 / 아무것도 안 함 / 다른 설계사
  2. 차별화 속성: 경쟁 대안 대비 뭐가 다른가?
     → 지속 관리 + AI 자동화 + 핀사이트랩스 데이터
  3. 가치: 그게 고객에게 왜 좋은가?
     → 내가 신경 안 써도 알아서 모이는 자산
  4. 타겟: 이 가치를 가장 원하는 사람은?
     → 바쁜 30대 맞벌이 (시간이 없다)
  5. 카테고리: 어떤 카테고리에 있는가?
     → "재무 관리 파트너" (설계사 아님)

상품별 포지셔닝:
  컨설팅 → "내 재무 담당자"
  교육콘텐츠 → "금융 문해력 키우기"
  핀사이트랩스 → "기관급 정보를 개인에게"
```

## JW Financial 상품 로드맵

```
현재 운영:
  ✅ 1:1 컨설팅 (포트폴리오 설계 + 관리)
  ✅ 핀사이트랩스 콘텐츠 (무료)

개발 중:
  □ 보험료 비교 시뮬레이터
  □ 교육 콘텐츠 패키지 (유료)
  □ 핀사이트랩스 프리미엄 구독

탐색:
  □ 그룹 컨설팅/세미나
  □ 중국어권 금융 입문 콘텐츠
```
