> JW Financial Consulting 실전 콘텐츠 제작 스킬
> 참조 프레임워크: Donald Miller (StoryBrand), Ann Handley (Everybody Writes), David Perell (Writing Online)
> ※ marketing.md가 "전략/채널/방향"이라면, 이 스킬은 "실제로 쓰는 법"

## 언제 이 스킬을 쓰는가

대장은 아래 키워드 포함 시 이 스킬을 부장에게 전달한다:
- `써줘`, `초안`, `작성해줘`, `글 써줘`
- `카드뉴스 본문`, `블로그 본문`, `스크립트`
- `카카오 메시지 본문`, `문자 초안`
- `이메일 초안`, `안내문`, `공지`
- `핀사이트랩스 리포트 본문`
- `고객에게 보낼 글`

> marketing.md와 같이 쓸 때:
> marketing.md → 어떤 채널에, 어떤 독자에게, 어떤 각도로
> content-creation.md → 실제로 어떻게 쓸 것인가

## FRAMEWORK 1 — StoryBrand (Donald Miller)

**핵심 원칙:** "고객이 주인공이다. 브랜드는 가이드다."

금융 콘텐츠의 가장 흔한 실수:
진우님(브랜드)이 주인공이 되는 것 → 독자가 공감 못 함

StoryBrand 7요소를 콘텐츠에 적용:

```
1. 주인공 (Hero) = 고객
   "30대 맞벌이 직장인인 당신"

2. 문제 (Problem)
   외부 문제: "노후 자금이 부족할 것 같다"
   내부 문제: "뭘 어떻게 해야 할지 모르겠다" ← 이게 더 중요
   철학적 문제: "열심히 일했는데 왜 노후가 불안한가"

3. 가이드 (Guide) = JW Financial
   공감: "그 불안, 당연합니다"
   권위: "5년간 100명 이상 설계해왔습니다"

4. 계획 (Plan)
   "이렇게 하면 됩니다 — 3단계"

5. 행동 촉구 (Call to Action)
   직접적: "지금 상담 신청하세요"
   과도기적: "무료 체크리스트 받아보세요"

6. 성공 (Success)
   "63세부터 매달 300만원, 걱정 없이"

7. 실패 회피 (Failure)
   "지금 시작하지 않으면 — 이렇게 됩니다"
```

### 콘텐츠별 StoryBrand 적용

**카드뉴스 구조:**
```
1컷: 문제 제시 (독자의 상황)
2~4컷: 문제의 원인/갭 설명 (내가 몰랐던 것)
5~6컷: 해결책/계획 제시
7컷: CTA (행동 촉구)
```

**블로그 포스팅 구조:**
```
도입: 독자 상황 공감 (주인공 = 독자)
전개: 문제의 본질 설명
해결: JW Financial 관점의 해답
마무리: 성공 그림 + CTA
```

## FRAMEWORK 2 — Everybody Writes (Ann Handley)

**핵심 원칙:** "좋은 글은 재능이 아니라 습관과 프로세스다"

### 황금률: TREAT 원칙

```
T — Truthful (진실되게)
  수치 과장 금지, 모르면 모른다고 쓰기
  "이 정도 수익이 가능합니다" (X) → "과거 데이터 기준입니다" (O)

R — Relevant (관련있게)
  독자의 현재 상황과 연결
  "30대 맞벌이라면 지금 이게 가장 중요합니다"

E — Empathetic (공감하게)
  판단하지 않기, 독자 편이 되기
  "아직 안 하셨다고 걱정 마세요"

A — Appropriate (적절하게)
  채널과 독자에 맞는 길이/톤/형식

T — Targeted (타겟 명확하게)
  한 명에게 쓰는 편지처럼
  "30대 맞벌이 직장인 김진수 씨"를 머릿속에 그리고 쓰기
```

### 문장 원칙 (JW Financial 스타일)

**문장 길이:**
```
카카오 메시지: 한 문장 15자 이내
카드뉴스 본문: 한 문장 20자 이내
블로그: 한 문장 30자 이내, 단락 3~4줄
```

**금지 표현 → 대체 표현:**
```
"~을 통해" → "~으로"
"~을 위한" → "~에 맞는"
"~에 대해서" → "~을"
"사실은" → (삭제)
"솔직히 말씀드리면" → (삭제)
"매우/굉장히/정말" → 구체적 숫자로 대체
"좋은" → 왜 좋은지로 대체
```

**첫 문장 공식:**
```
질문형: "연말정산 때 세금 더 낸 적 있으세요?"
상황형: "맞벌이 부부가 자산을 합치면 안 되는 이유가 있습니다"
숫자형: "직장인 10명 중 7명이 IRP를 잘못 쓰고 있습니다"
역설형: "보험이 필요 없는 유일한 경우"
```

## FRAMEWORK 3 — Writing Online (David Perell)

**핵심 원칙:** "온라인 글쓰기는 신뢰를 쌓는 가장 빠른 방법이다"

Perell의 콘텐츠 레버리지 개념:
한 번 잘 쓴 글 → 여러 채널로 재활용 → 시간 대비 임팩트 극대화

### 콘텐츠 원자 → 분자 전략 (JW Financial 적용)

```
핵심 아이디어 1개 (원자)
        ↓
블로그 포스팅 (분자 1) — 1,000자 심화 버전
카드뉴스 7컷 (분자 2) — 시각화 요약 버전
카카오 메시지 (분자 3) — 3줄 핵심 버전
숏폼 스크립트 (분자 4) — 60초 말하기 버전
핀사이트랩스 리포트 (분자 5) — 분석 심화 버전
```

**예시: "IRP 세액공제" 원자 하나로:**
```
블로그: "IRP 세액공제 완전정복 — 얼마나 돌려받나"
카드뉴스: "IRP 세액공제, 5컷으로 끝내기"
카카오 메시지: "연말정산 D-30, IRP 한도 채우셨나요?"
숏폼: "IRP로 66만원 돌려받는 법 (30초)"
```

### Perell의 개인 관점 원칙

```
정보 콘텐츠 (누구나 씀)
  "IRP란 개인형퇴직연금입니다"

관점 콘텐츠 (JW Financial만 씀)
  "IRP, 5년 동안 설계해보니 이렇게 쓰는 사람만 진짜 혜택 봅니다"
```

→ 부장은 항상 **관점 콘텐츠** 지향
→ 단순 정보 나열 금지, 진우님의 5년 경험/관점 반드시 포함

## 콘텐츠 유형별 실전 작성 가이드

### 카드뉴스 (7컷 표준)
```
컷 1 — 후킹 제목 (스크롤 멈추게)
  공식: [숫자/역설/공감] + [타겟 상황]
  예: "맞벌이인데 왜 노후가 불안할까요?"

컷 2 — 문제 공감
  "이런 상황이시죠?" — 독자 상황 정확히 묘사

컷 3~5 — 핵심 내용 (1컷 1메시지)
  컷당 헤드라인 1줄 + 설명 2줄 이내
  비유/숫자 반드시 포함

컷 6 — 해결책/정리
  "그래서 이렇게 하면 됩니다"

컷 7 — CTA
  "더 알고 싶으시면 → [링크]"
  또는 "무료 상담 신청하기"
```

### 블로그 포스팅 (SEO + 가독성)
```
제목: 메인 키워드 포함, 30자 이내
  예: "30대 직장인 IRP 세액공제 완전정복"

도입부 (150자):
  독자 상황 공감 → 이 글에서 얻을 것 예고

목차 (3~5개):
  소제목마다 키워드 포함

본문 단락 규칙:
  - 단락 4줄 이내
  - 소제목마다 요약 1줄
  - 숫자/표/예시 최소 1개/단락

마무리:
  핵심 3줄 요약 + CTA
```

### 카카오 메시지 (개인 발송용)
```
형식:
  [이름]님, (1줄 공감/계절인사)
  [핵심 메시지 1줄]
  [행동 1개] → [링크 or 답장 유도]

원칙:
  - 전체 5줄 이내
  - 이름 반드시 호명
  - 이모지 최대 2개
  - CTA 1개만
  - 판매 느낌 금지

예시:
  진수님, 어느덧 4월이네요 🌸
  연금저축 납입 한도 아직 여유 있으실 거예요.
  이번 달 안에 채우시면 5월 환급 때 잡힙니다.
  5분이면 확인 가능해요 → [링크]
```

### 핀사이트랩스 리포트 (분석 심화)
```
헤더: [날짜] [주제] — 핀사이트랩스 Weekly/Monthly

구조:
  📌 한줄요약 (이 리포트의 핵심 결론 먼저)
  
  1. 현황 — 이번 주/달 주요 데이터
  2. 해석 — 왜 이런 결과인가 (인과관계)
  3. 사이클 — 지금 어디쯤인가
  4. 시사점 — 개인 투자자에게 의미
  5. 액션 — 할 것 / 볼 것 / 피할 것

톤: 기관투자자 시각, 감정 배제, 데이터 근거
길이: 800~1,200자
```

## 콘텐츠 품질 체크리스트 (부장 자가검토)

작업 완료 전 반드시 확인:

```
[ ] 첫 문장이 독자를 멈추게 하는가?
[ ] 독자(고객)가 주인공인가? (브랜드 자랑 X)
[ ] 비유나 구체적 숫자가 최소 1개 있는가?
[ ] 판매 언어가 없는가?
[ ] CTA가 정확히 1개인가?
[ ] 모바일에서 읽기 편한 길이인가?
[ ] 진우님 관점/경험이 담겼는가?
[ ] ⚠️ 확인필요 표기가 필요한 수치는 표기했는가?
```
