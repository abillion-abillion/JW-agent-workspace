## JW Financial 핵심 법률 환경

```
적용 법률:
  □ 자본시장과 금융투자업에 관한 법률 (자본시장법)
  □ 보험업법
  □ 금융소비자 보호에 관한 법률 (금소법)
  □ 개인정보 보호법
  □ 전자상거래법 (온라인 서비스 관련)

주요 규제 포인트:
  □ 투자 권유 시 적합성 원칙 준수
  □ 설명의무 — 상품 위험/수익 구조 명확히 설명
  □ 불공정 영업행위 금지
  □ 광고 규제 — 수익률 보장 표현 금지
  □ 개인정보 수집/보관/파기 절차
```

## FRAMEWORK 1 — Alan Dershowitz (리스크 프레이밍)

**핵심 원칙:** "최악의 시나리오를 먼저 생각하고, 그것을 피하는 구조를 만들어라"

### 법적 리스크 사전 진단

```
모든 새 업무/서비스 시작 전 체크:

1. 최악의 시나리오는?
   "이 행동이 문제가 된다면 어떤 법률 위반인가?"

2. 실제 발생 가능성은?
   높음 / 중간 / 낮음

3. 발생 시 피해 규모는?
   금전적 손실 / 브랜드 훼손 / 라이선스 취소

4. 예방 방법은?
   계약서 / 동의서 / 설명 기록 / 면책 조항
```

### JW Financial 주요 리스크 맵

```
🔴 고위험 (즉시 대응):
  - 수익률 보장 표현 (광고/콘텐츠)
  - 적합성 미확인 투자 권유
  - 개인정보 무단 수집/공유

🟡 중위험 (관리 필요):
  - 계약서 없는 컨설팅
  - 녹취 동의 미취득
  - 광고에 과거 성과 단독 표시

🟢 저위험 (모니터링):
  - 법령 개정 동향
  - 금융당국 제재 사례 업데이트
```

## FRAMEWORK 2 — Bryan Garner (계약서/문서)

**핵심 원칙:** "좋은 계약서는 분쟁을 막는다. 나쁜 계약서는 분쟁을 만든다"

### JW Financial 필수 문서 목록

```
고객 관련:
  □ 컨설팅 계약서 (서비스 범위, 비용, 해지 조건)
  □ 개인정보 수집·이용 동의서
  □ 상담 녹취 동의서 (Clova Note 사용 시)
  □ 투자성향 확인서 (금융상품 권유 전)
  □ 상품 설명 확인서 (설명의무 이행 증거)

콘텐츠/마케팅 관련:
  □ 수익률 표시 기준 체크리스트
  □ 광고 문구 컴플라이언스 체크
  □ 비식별 사례 활용 동의 (고객 사례 콘텐츠 사용 시)

팀/동료 관련:
  □ 근로계약서 or 용역계약서
  □ 비밀유지 협약 (NDA) — 고객 정보 보호
  □ 수수료 배분 기준 문서
```

### 계약서 작성 원칙 (Garner)

```
명확성:
  → 애매한 표현 제거 ("적절한 기간" → "30일 이내")
  → 수치로 명시 ("상당한 비용" → "월 50만원 이내")

완전성:
  → 예외 상황까지 기재 (해지, 분쟁, 불가항력)
  → 당사자 의무 양쪽 모두 명시

실행 가능성:
  → 현실적으로 이행 가능한 조건만
  → 과도한 위약금 조항은 무효 가능성
```

## FRAMEWORK 3 — Richard Susskind (법률 테크/미래)

**핵심 원칙:** "반복되는 법률 업무는 시스템화하라. 사람은 판단에만 집중"

### JW Financial 컴플라이언스 자동화

```
자동화 가능한 컴플라이언스:
  □ 콘텐츠 발행 전 금지 표현 자동 체크
      → "보장", "확정", "반드시" 등 키워드 필터
  □ 개인정보 보유 기간 만료 알림
  □ 계약 만료 사전 알림
  □ 법령 개정 모니터링 (금융위/금감원 공지)

사람이 직접 판단해야 하는 것:
  □ 계약서 최종 검토
  □ 고객 분쟁 대응
  □ 새로운 법률 해석
  □ 규제 당국 대응
```

### 금융 광고·콘텐츠 컴플라이언스 체크리스트

```
발행 전 반드시 확인:

수익률 관련:
  □ "보장", "확정", "반드시" 표현 없는가
  □ 과거 수익률 표시 시 "과거 성과가 미래를 보장하지 않음" 명시
  □ 예상 수익은 "예시" 또는 "시뮬레이션" 명확히 표기

상품 설명 관련:
  □ 위험 요소 함께 기재 (수익만 강조 금지)
  □ 원금 손실 가능성 명시 (해당 상품)
  □ 특정 상품 추천 시 적합성 근거

개인정보 관련:
  □ 비식별 고객 사례 — 특정 불가한 수준인가
  □ 동의받지 않은 개인정보 콘텐츠 미포함

일반:
  □ 금융당국 등록 정보 표시 (필요 시)
  □ 투자 권유 여부 명확히 구분
```

## 분쟁/민원 대응 SOP

```
Step 1. 즉시 기록
  - 민원 내용, 날짜, 채널 기록
  - 관련 계약서/동의서/상담 기록 즉시 확보

Step 2. 감정 분리
  - 사실 확인 전 사과 or 인정 금지
  - "확인 후 연락드리겠습니다" 먼저

Step 3. 사실 파악
  - 실제로 무슨 일이 있었는가
  - 계약서/동의서 기준으로 의무 이행 여부 확인

Step 4. 대응 방향 결정
  - 정당한 민원 → 신속 해결 + 재발 방지
  - 부당한 민원 → 계약서 근거로 정중히 대응
  - 법적 분쟁 가능성 → 즉시 전문 변호사 상담

Step 5. 사후 문서화
  - 대응 내용, 결과 기록
  - 재발 방지 SOP 업데이트
```
