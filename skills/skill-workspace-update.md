# 스킬: 워크스페이스 업데이트 (skill-workspace-update)

> 진우님이 Telegram으로 말하면 GitHub 파일을 직접 수정

---

## 사용법

### 스킬 추가
```
"CS팀 카카오 메시지에 노쇼 리스케줄 템플릿 추가해줘"
"영업팀 포트폴리오 북에 브라질 국채 섹션 추가해줘"
"소개 요청 스크립트에 크몽 고객용 버전 추가해줘"
```

### 스킬 수정
```
"이탈방지 D+1 메시지 톤 좀 더 캐주얼하게 바꿔줘"
"온보딩 D+30 메시지에 시뮬레이터 링크 추가해줘"
```

### 메모리 업데이트
```
"ongoing에 보험료 시뮬레이터 프로젝트 시작했다고 추가해줘"
"오늘 업무일지 저장해줘 - [내용]"
```

### 팀 구조 확인
```
"영업팀 스킬 목록 보여줘"
"CS팀 담당 업무 뭐야?"
"전체 워크스페이스 구조 보여줘"
```

---

## 업데이트 원칙

1. 수정 전 현재 내용 먼저 보여주고 확인 요청
2. 수정 완료 후 변경된 내용 요약 보고
3. 삭제는 반드시 진우님 명시적 확인 후 실행
4. 모든 업데이트에 날짜 기록
